# Model Management

When you work with multiple providers and models, it is often desirable to manage them in a central place
and access the models through simple string ids.

The Vercel AI SDK provides the `ModelRegistry` class for this purpose.
You can register either individual models or providers with the registry and access them by their ids.
If you register a provider, the provider id will become the prefix of the model id:
`providerId:modelId`.

### Setup (Example)

<Note>
  It is common to keep the registry setup in a separate file and import it where
  needed.
</Note>

```ts highlight={"3,5,8-13,16-19,27-30"} filename={"registry.ts"}
import { anthropic } from '@ai-sdk/anthropic';
import { createOpenAI } from '@ai-sdk/openai';
import { ModelRegistry } from 'ai';

export const registry = new ModelRegistry();

// register provide with prefix and custom setup
registry.registerLanguageModelProvider({
  id: 'openai',
  provider: createOpenAI({
    apiKey: process.env.OPENAI_API_KEY,
  }),
});

// register provide with prefix and default setup
registry.registerLanguageModelProvider({
  id: 'anthropic',
  provider: anthropic,
});

// register specific model with id
const groq = createOpenAI({
  baseURL: 'https://api.groq.com/openai/v1',
  apiKey: process.env.GROQ_API_KEY,
});

registry.registerLanguageModel({
  id: 'llama3',
  model: groq('llama3-8b-8192'),
});
```

### Usage (Example)

```ts highlight={"5"}
import { generateText } from 'ai';
import { registry } from './registry';

const { text } = await generateText({
  model: registry.languageModel('openai:gpt-4-turbo'),
  prompt: 'Invent a new holiday and describe its traditions.',
});
```
